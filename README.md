2. Conceptual Framework & Background

2.1 What is Prompt Engineering?
Prompt engineering is the systematic practice of designing, refining, and optimizing input instructions to
AI language models to produce desired outputs reliably. It bridges the gap between human intent and
AI behavior. Effective prompt engineering considers context, specificity, format instructions, role
assignment, and constraint definition.

8. Reflection Notes & Lessons Learned

8.1 What Worked Well
• Role-based prompts dramatically improved code quality – assigning 'senior Python developer'
role produced more production-ready code
• Format instructions were critical: requesting table format produced structured, comparable
outputs from both AI tools
• Adding library specifications ('use requests library') eliminated ambiguity and prevented AI from
choosing less appropriate alternatives
• Breaking the task into 3 stages mirrored real software development workflows and made
prompts more focused
• Claude was more responsive to object-oriented design hints; ChatGPT responded better to
simple function-based prompts

9. Conclusion & Next Steps
9.1 Summary of Findings
This exercise successfully demonstrated the power of systematic prompt engineering for AI-assisted
code development. The key findings were:
1. Prompt specificity is the single most impactful factor – vague prompts produced usable but
suboptimal code; refined prompts produced production-quality code
2. Both ChatGPT and Claude produced correct Python code, but with measurably different styles
and architectural preferences
3. Claude scored 46/50 vs ChatGPT's 37/50 in code quality for this specific task, particularly
excelling in structure and extensibility
4. The RACE framework (Role, Action, Context, Expectation) proved highly effective as a prompt
architecture template
5. Comparing two AI tools revealed unique strengths: ChatGPT for quick prototypes, Claude for
production-ready architectures
